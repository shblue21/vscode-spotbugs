# Support

Thanks for using SpotBugs for VS Code.

## Getting Help

Use GitHub Issues for bug reports, feature requests, and general questions:

https://github.com/shblue21/vscode-spotbugs/issues

Before opening a new issue:

- Search existing issues for similar reports.
- Update to the latest SpotBugs extension version.
- Confirm the workspace has Java 11 or later and the `redhat.java` extension installed.
- Confirm the Java project can be imported and built by VS Code.

## Bug Reports

When reporting a bug, include:

- OS and VS Code version.
- SpotBugs extension version.
- Java version.
- Project type, such as Maven or Gradle.
- The command you ran, such as `SpotBugs: Analyze this workspace`.
- Error messages, relevant logs, screenshots, or a small sample project.

## Feature Requests

Open an issue and describe the Java workflow you want to improve. Include any relevant SpotBugs options, filter files, SARIF usage, or project setup details.

## Security-Sensitive Reports

Do not open a public issue for vulnerabilities or reports that include private source code, credentials, or other sensitive data.

Email the maintainer at shblue21@naver.com with a concise description and the minimum details needed to reproduce or understand the issue.

## Support Scope

Support is best effort. Issues with clear reproduction steps, environment details, and logs are easier to investigate.
